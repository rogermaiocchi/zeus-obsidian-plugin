# /graphify

Turn any folder of files into a navigable knowledge graph with community detection, an honest audit trail, and three outputs: interactive HTML, GraphRAG-ready JSON, and a plain-language GRAPH_REPORT.md.

## Usage

```
/graphify                                             # full pipeline on current directory → Obsidian vault
/graphify <path>                                      # full pipeline on specific path
/graphify https://github.com/<owner>/<repo>           # clone repo then run full pipeline on it
/graphify https://github.com/<owner>/<repo> --branch <branch>  # clone a specific branch
/graphify <url1> <url2> ...                           # clone multiple repos, build each, merge into one cross-repo graph
/graphify <path> --mode deep                          # thorough extraction, richer INFERRED edges
/graphify <path> --update                             # incremental - re-extract only new/changed files
/graphify <path> --directed                            # build directed graph (preserves edge direction: source→target)
/graphify <path> --whisper-model medium                # use a larger Whisper model for better transcription accuracy
/graphify <path> --cluster-only                       # rerun clustering on existing graph
/graphify <path> --no-viz                             # skip visualization, just report + JSON
/graphify <path> --html                               # (HTML is generated by default - this flag is a no-op)
/graphify <path> --svg                                # also export graph.svg (embeds in Notion, GitHub)
/graphify <path> --graphml                            # export graph.graphml (Gephi, yEd)
/graphify <path> --neo4j                              # generate graphify-out/cypher.txt for Neo4j
/graphify <path> --neo4j-push bolt://localhost:7687   # push directly to Neo4j
/graphify <path> --mcp                                # start MCP stdio server for agent access
/graphify <path> --watch                              # watch folder, auto-rebuild on code changes (no LLM needed)
/graphify <path> --wiki                               # build agent-crawlable wiki (index.md + one article per community)
/graphify <path> --obsidian --obsidian-dir ~/vaults/my-project  # write vault to custom path (e.g. existing vault)
/graphify add <url>                                   # fetch URL, save to ./raw, update graph
/graphify add <url> --author "Name"                   # tag who wrote it
/graphify add <url> --contributor "Name"              # tag who added it to the corpus
/graphify query "<question>"                          # BFS traversal - broad context
/graphify query "<question>" --dfs                    # DFS - trace a specific path
/graphify query "<question>" --budget 1500            # cap answer at N tokens
/graphify path "AuthModule" "Database"                # shortest path between two concepts
/graphify explain "SwinTransformer"                   # plain-language explanation of a node
```

## What graphify is for

Drop any folder of code, docs, papers, images, or video into graphify and get a queryable knowledge graph. Persistent across sessions, honest audit trail (EXTRACTED/INFERRED/AMBIGUOUS), community detection surfaces cross-document connections you wouldn't think to ask about.

### Passo 2 — Retrieval com Decay Scoring (Tsinghua MemoryBank + Stanford)

Abrir `MEMORY.md` (ja esta no contexto). Filtrar por tags do perfil detectado.
Ranquear usando formula composta:

```
score = (0.4 * tag_relevance) + (0.3 * imp/5) + (0.3 * recency_score)

Onde:
  tag_relevance = 1.0 se tag faz match direto, 0.5 se tag pai faz match
  imp = campo imp: do MEMORY.md (1-5), normalizado para 0-1
  recency_score = max(0, 1 - dias_desde_modified/90)
    → arquivo acessado hoje = 1.0
    → arquivo de 30 dias atras = 0.67
    → arquivo de 90+ dias = 0.0 (candidato a archival/prune)
```

```
REGRA: Carregar NO MAXIMO 3-5 arquivos por sessao (score > 0.4).
REGRA: Itens [CORE] so carregam se NENHUM perfil especifico fizer match.
REGRA: Itens #desativado NUNCA carregam automaticamente.
REGRA: Ao carregar um arquivo, atualizar seu frontmatter:
       modified: (data de hoje)
       Isso renova a recency e evita decay excessivo.
```

Exemplo de filtragem (SQL-like com scoring):
```sql
SELECT file, (0.4*tag_match + 0.3*imp/5 + 0.3*recency) AS score
FROM MEMORY
WHERE tags INTERSECT (#bot)
  AND tier != 'archival'
ORDER BY score DESC
LIMIT 3;
```

## Tiers de Memoria (Zhejiang WISE + USTC MemoryLLM)

Cada arquivo de memoria pertence a um tier:

REGRA: Tiers sao determinados pelo campo imp: no MEMORY.md:
  imp:5 = Imutavel ou Core (sempre/quase-sempre relevante)

## Retrieval Progressivo em 3 Niveis (ByteRover Context Tree)

Resolver queries no nivel mais raso possivel antes de carregar mais contexto:

```
Nivel 1 — MEMORY.md index (ja no contexto, ~70 linhas)
  → Responde: "que projetos existem?", "qual tag usar?", "onde fica X?"
  → Se resolve aqui, NAO carregar nenhum arquivo.

Nivel 2 — Carregar 1-3 arquivos por tag match (score > 0.5)
  → Responde: "qual o estado do bot?", "como funciona o vault?"
  → Ler SOMENTE os arquivos top-ranked.

Nivel 3 — Carregar arquivos auxiliares + ponteiros para vault
  → Responde: "preciso dos detalhes da arquitetura X"
  → Ler ponteiros em project_vault_state.md para arquivos do vault
  → Ou usar Agent tool para pesquisa profunda (nao infla contexto principal)
```

```
REGRA: Maioria das queries resolve no Nivel 1 ou 2.
REGRA: Nivel 3 so com necessidade confirmada.
REGRA: Se Nivel 3 envolver leitura de >3 arquivos, usar Agent tool
       (subagente tem contexto proprio, protege o principal).
```

## Confidence Tags (Graphify Pattern)

Cada memoria carrega um nivel de confianca no frontmatter:

| Confidence | Significado | Acao no retrieval |
|------------|-------------|-------------------|
| `verified` | Usuario confirmou explicitamente ou dado extraido de fonte primaria | Prioridade maxima |
| `inferred` | Extraido de contexto de sessao, nao confirmado | Usar mas validar se critico |
| `stale` | >60 dias sem acesso, informacao pode estar desatualizada | Verificar antes de usar |

```
REGRA: Ao criar nova memoria, marcar confidence: inferred
REGRA: Quando usuario confirma informacao → promover para verified
REGRA: Durante /consolidate-memory, memorias com recency < 0.3 → stale
REGRA: Memorias stale + imp:1 → candidatas a exclusao
```

   No iPhone/iPad: no plugin, ligar **Permitir fallback remoto para mesh peers**, adicionar `http://<IP-ou-host-Tailscale-do-Mac>:2223` em **Daemon mesh peers**, colar o mesmo token em **Token do daemon remoto**, e usar **Forçar redescoberta de daemon agora**.  
   Garantir Tailscale conectado nos dois devices e que a porta `2223` esteja alcançável no Mac.

Os resultados empíricos que você passou são coerentes com o código: remoto sem token `401`, remoto com token `200`, `/v1/cmd` remoto `403`, loopback `/v1/cmd` `200`.
